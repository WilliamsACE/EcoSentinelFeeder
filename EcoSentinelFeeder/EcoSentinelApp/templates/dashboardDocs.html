Dashboard EcoSentinel

Todo lo que necesitas saber para entender, mantener y depurar el JavaScript del dashboard. Cubre Chart.js, Leaflet, el sistema de polling con Django, notificaciones, configuración persistente y el sistema responsive.

Chart.js 4.4 Leaflet 1.9 Django REST localStorage sessionStorage setInterval polling

🏗️ Arquitectura general

El dashboard es un archivo HTML único (sin build system, sin frameworks). Todo el JavaScript vive en un solo bloque <script> al final del <body>. No hay módulos ES6, no hay imports — es JavaScript vanilla puro, compatible con cualquier navegador moderno.

📦 Librerías externas

Se cargan vía CDN en el <head>. Si cambia la URL del CDN o la versión, actualiza los <script src="..."> correspondientes.

⚠ Sin conexión a internet
Si el dashboard se abre sin internet, Leaflet y Chart.js fallarán silenciosamente y los mapas / gráficas no se renderizarán. Para uso offline, descarga las librerías y cámbialas a rutas locales.

💡 Por qué se inyectan via JS
Para que los skeletons sean opcionales y no ensucie el <style> principal. Si en el futuro quieres quitarlos, simplemente elimina la IIFE y todas las referencias a clases .skel-* en el HTML.

🧭 Navegación de secciones

El dashboard es una SPA falsa: todo el HTML de todas las secciones existe en el DOM desde el inicio, pero solo una tiene display:block a la vez. No hay routing real.

switchSection(id) sync

Muestra la sección con id="section-{id}" y oculta todas las demás. Actualiza el título del topbar y marca el nav-item activo.

Efectos secundarios importantes: si id === "mapa" llama a initMapFull() con 60ms de delay (para que el elemento sea visible antes de que Leaflet mida su tamaño). Si id === "donaciones" llama a syncDonaciones().

// Cómo añadir una nueva sección:
// 1. Agrega el ID al objeto de títulos:
const SECTION_TITLES = {
  mi-nueva-seccion: "Mi nueva sección",
  ...
};

// 2. Crea el HTML de la sección:
// <section id="section-mi-nueva-seccion" class="page-section">

// 3. Crea el nav-item en el sidebar:
// <a class="nav-item" data-section="mi-nueva-seccion">

// Listo. El event listener ya lo conecta automáticamente:
document.querySelectorAll('.nav-item[data-section]').forEach(el => {
  el.addEventListener('click', e => {
    e.preventDefault();
    switchSection(el.dataset.section);
  });
});
⚠ En móvil, switchSection está envuelta
Al final del script, window.switchSection es reemplazada por una versión que también llama a closeMobileSidebar(). Si llamas a switchSection() desde cualquier punto, el sidebar se cierra automáticamente en móvil.

🔔 Sistema de notificaciones

Las notificaciones se almacenan en notifStore, un array en memoria. No persiste entre recargas. Cada vez que el array cambia, se llama a renderNotifications() que repinta todos los contenedores de alertas.

Estructura de una notificación

{
  type:        "danger" | "warn" | "ok",  // determina el color del punto
  title:       "ESF-001 — Niveles bajos",
  description: "Batería: 12% · Dog food: 8%",
  time:        "14:32",
  location:   "Del Valle"
}

Funciones disponibles

addNotification(notif)sync

Agrega al inicio del array y llama a renderNotifications(). Las notificaciones duplicadas (mismo título) se filtran en syncDashboard() antes de llamar a esta función.

clearNotifications()sync

Vacía el array y repinta. Se llama desde el botón "Marcar todas".

renderNotifications()sync

Repinta simultáneamente los contenedores #notif-list-resumen y #notif-list-alertas. También actualiza los badges de conteo en el topbar, en el sidebar y en la campana. Ambos contenedores ven siempre los mismos datos.

Evitar duplicados
En syncDashboard(), antes de llamar a addNotification(), se verifica si ya existe una notificación con el mismo título: notifStore.find(n => n.title === a.title). Así el polling no genera alertas repetidas cada 30 segundos.

🎨 Funciones de render

Estas funciones reciben datos del API y los pintan en el DOM. Son puras respecto al DOM: leen datos, escriben HTML. No hacen peticiones de red.

renderFeeders(feeders)

Recibe el array de feeders del API y actualiza seis áreas del DOM en una sola llamada:

ÁreaElemento DOMDescripción
KPIs#kpi-animals, #kpi-food, #kpi-online, #kpi-alertsLos 4 números grandes del resumen.
KPI subs#kpi-animals-sub, etc.Texto descriptivo debajo de cada KPI.
Sidebar list#sidebar-feeder-listLa lista de feeders activos en el menú lateral.
Estado rápido#feeder-quick-listLas tarjetas con mini-gauges de barra del resumen.
Detail grid#feeders-detail-gridLas tarjetas grandes con gauges SVG del resumen.

También llama a _refreshMapPins(feeders) para actualizar los marcadores del mapa, y a renderFeederCards(feeders) para la sección "Todos los feeders".

Cálculo de los KPIs

// Animales: suma de perros + gatos de todos los feeders hoy
animals += (f.dogsToday || 0) + (f.catsToday || 0);

// Alimento: aprox 85g por perro, 60g por gato
kg += ((f.dogsToday || 0) * 85 + (f.catsToday || 0) * 60) / 1000;

// Feeders online vs alertas
if (f.status === "online") online++;
else if (f.status === "warn") alerts++; // "offline" no cuenta en alertas

Gauges SVG — cómo funciona el círculo

// El círculo SVG usa stroke-dasharray + stroke-dashoffset para el progreso
// Circunferencia = 2 * π * r = 2 * 3.14159 * 26 ≈ 163.4
const pct = (v) => Math.round(163.4 * (1 - v / 100));

// Ejemplo: batería al 75% → dashoffset = 163.4 * 0.25 = 40.85
// dashoffset=0 → círculo completo. dashoffset=163.4 → círculo vacío.
💡 Colores de los gauges
El color del trazo cambia según el valor: rojo si < 20%, naranja si < 40%, verde si ≥ 40%. Excepción: el gauge de agua siempre es azul claro (#a8c8e8).

addHistoryEvent(ev)

Inserta una fila nueva al principio de las dos tablas de historial (#history-body y #history-body-extended). Limita a 20 filas eliminando la última cuando se supera ese límite.

Estructura del evento

{
  time:       "14:32",
  feederId:  "ESF-001",
  feederName:"Del Valle",
  species:   "perro" | "gato" | "alerta", // clase CSS de la badge
  emoji:     "🐕",
  label:     "Perro",
  grams:     85,          // 0 si no aplica
  confidence:94          // porcentaje de precisión IA, 0-100
}

updateCharts(dispensa, species, trend)

Actualiza los tres charts de Chart.js con datos nuevos. Cada parámetro puede ser null o undefined — la función verifica antes de actualizar.

🔄 Sincronización con Django

El dashboard usa polling HTTP (no WebSockets). Cada N segundos hace un fetch() al API de Django y actualiza el DOM. Los intervalos son configurables desde la sección Configuración.

syncDashboard()async

Llama a GET /api/dashboard/. Espera un JSON con la estructura:

{
  feeders: [ /* array de feeders */ ],
  charts: {
    dispensa: { labels: [], dogs: [], cats: [] },
    species:  { dogs: 84, cats: 58 },
    trend:    { labels: [], dogs: [], cats: [] }
  },
  alerts: [ /* opcional — array de notifs */ ]
}

Timeout: 10 segundos (AbortSignal.timeout(10000)). Si falla, muestra el error-banner 7 segundos.

syncHistory()async

Llama a GET /api/history/?limit=20. Espera:

{ events: [ /* array de eventos para addHistoryEvent() */ ] }

Timeout: 8 segundos. Los errores solo van a consola, no al banner de error.

refreshData()sync

Llama a ambas funciones en paralelo con Promise.all(). Desactiva el botón "Actualizar" durante la carga.

Variables de los intervalos de polling

let _syncDashInterval, _syncHistInterval;

// Se inicializan con los valores de localStorage (o 30s / 15s por defecto)
const _sd0 = parseInt(localStorage.getItem("cfg_sync_d") || "30");
const _sh0 = parseInt(localStorage.getItem("cfg_sync_h") || "15");

// guardarIntervalos() los reinicia con los nuevos valores:
clearInterval(_syncDashInterval);
clearInterval(_syncHistInterval);
_syncDashInterval = setInterval(syncDashboard, sd * 1000);
_syncHistInterval = setInterval(syncHistory,   sh * 1000);
⚠ CSRF en peticiones POST
La función executeDelete() hace un POST y necesita el token CSRF de Django. Lo obtiene con _getCookie("csrftoken") que lee la cookie que Django pone automáticamente. Si Django tiene CSRF_COOKIE_HTTPONLY = True en settings, esto fallará — esa opción debe estar en False.

Estado del sync en el topbar

// _setSyncStatus cambia el punto de color y el texto
_setSyncStatus("loading", "Sincronizando…");  // punto naranja pulsante
_setSyncStatus("ok",      "Última sync: 14:32");// punto verde estático
_setSyncStatus("error",   "Error de conexión");  // punto rojo fijo

Hay dos instancias independientes de Leaflet: una en el resumen (_mapSmall) y otra en la sección Mapa completo (_mapFull). Comparten los mismos marcadores lógicamente pero son instancias distintas.

Funciones del mapa

_mkIcon(color)privada

Crea un L.divIcon personalizado: círculo blanco con borde del color del status y emoji 🐾 dentro. Los íconos SVG de Leaflet por defecto son feos — esta función los reemplaza.

_mkPopup(f)privada

Genera el HTML del popup que aparece al hacer clic en un pin. Muestra ID, nombre, batería, comida perros/gatos, agua y estado. Los valores bajos (<20%) se muestran en rojo.

_refreshMapPins(feeders)privada

Elimina todos los markers actuales y los recrea con los datos nuevos. Se llama desde renderFeeders(). Opera en ambas instancias si están inicializadas.

⚠ Problema clásico de Leaflet: mapa gris
Si el mapa aparece gris o incompleto, es porque Leaflet se inicializó cuando el elemento estaba oculto (display:none). Por eso switchSection("mapa") espera 60ms con setTimeout antes de llamar a initMapFull(). Si ves el mapa gris, aumenta ese delay a 100–200ms.
🚨 z-index y el sidebar móvil
Leaflet crea elementos con z-index hasta ~1000. El sidebar móvil usa z-index: 1100 y el overlay z-index: 1099 para garantizar que el sidebar aparezca encima del mapa en pantallas pequeñas.

🔍 Filtros de historial

filterHistory(btn, tbodyId)sync

Recibe el botón clickeado y el ID del <tbody> a filtrar. Muestra u oculta filas según la clase de la badge .ht-species dentro de cada <tr>.

Para filtro "todos" muestra todo. Para "perro", "gato" o "alerta", verifica si la badge tiene esa clase CSS.

// La función busca el botón activo dentro del grupo hermano más cercano:
btn.closest(".history-filters, .card-header")
   .querySelectorAll(".hf-btn[data-filter]")
   .forEach(b => b.classList.remove("active"));

💡 Botones 7D/30D/90D
Esos botones no tienen data-filter, por eso tienen su propio listener con el selector .hf-btn:not([data-filter]). Solo cambian el estilo activo — la lógica de cargar 30 o 90 días de datos todavía no está implementada (queda pendiente conectarla al API).

Los valores de cfg_bat, cfg_water, etc. se guardan en localStorage pero actualmente no se usan en la lógica de alertas del JS. Las alertas se generan del lado del servidor Django. Estos valores están pensados para ser enviados al backend en una futura implementación.

// o limpiar todo de golpe:
localStorage.clear();

🗑️ Modal — borrar todos los datos

El modal de confirmación requiere que el usuario escriba literalmente "aceptar" para habilitar el botón de borrado. Hace un POST /api/data/delete-all/ con el token CSRF.

FunciónQué hace
openDeleteModal()Muestra el modal (display:flex), resetea el input y enfoca el campo.
closeDeleteModal()Oculta el modal. También se dispara al clic en el backdrop o con Escape.
checkDeleteInput()Se llama en cada oninput. Habilita el botón solo si el valor es exactamente "aceptar".
executeDelete()Hace el POST, limpia el DOM (tablas + notificaciones) y llama syncDashboard() + syncHistory() para recargar datos frescos.
_getCookie(name)Utilidad para leer cookies del navegador. Se usa para obtener el csrftoken de Django.

📱 Sidebar móvil

toggleMobileSidebar()sync

Agrega/quita la clase .open al sidebar y al overlay. Cuando el sidebar está abierto, bloquea el scroll del body con document.body.style.overflow = "hidden".

closeMobileSidebar()sync

Cierra el sidebar. Se llama desde: el overlay (clic fuera), cada switchSection() (envuelta), y puede llamarse manualmente.

// Al final del script, switchSection original se reemplaza:

  window.switchSection = function(id) {

  };

🖥️ Banner de recomendación móvil

Aparece en la parte inferior como un toast flotante al entrar desde un dispositivo móvil. Se detecta por ancho de pantalla (window.innerWidth ≤ 768) o User Agent.

closeMobileBanner()sync

Oculta el banner con animación fade+slide. Guarda en sessionStorage la clave mobile_banner_dismissed = "1" para que no vuelva a aparecer en la misma sesión (desaparece al cerrar el tab, pero no al recargar dentro de la misma sesión).

// La IIFE que decide si mostrar el banner:
(function showMobileBannerIfNeeded() {
  const isMobile = window.innerWidth <= 768 ||
    /Android|iPhone|iPad|iPod|Mobile/i.test(navigator.userAgent);
  if (!isMobile) return;

  let dismissed = false;
  try { dismissed = sessionStorage.getItem("mobile_banner_dismissed") === "1"; } 
  catch(e) {}
  if (dismissed) return;

  setTimeout(() => { banner.style.display = "flex"; }, 800); // delay 800ms
})();
💡 Forzar que aparezca en desarrollo
En la consola del navegador: sessionStorage.removeItem("mobile_banner_dismissed") y recarga la página con el DevTools en modo móvil.

🚨 Fallos comunes y cómo debuggear

El mapa aparece gris o incompleto

Causa: Leaflet se inicializó cuando el elemento estaba oculto.
Fix: Aumenta el delay en switchSection(): setTimeout(initMapFull, 150).
Fix alternativo: Abre DevTools → consola → escribe _mapFull.invalidateSize().

Los charts no se actualizan

Causa: El API devuelve arrays vacíos o la estructura del JSON no coincide.
Debug: En consola: dispensingChart.data para ver los datos actuales.
Fix: Verifica que /api/dashboard/ devuelva charts.dispensa.labels con elementos.

Las notificaciones se duplican

Causa: El API está mandando alertas con títulos distintos cada vez.
Debug: En consola: notifStore para ver el array completo.
Fix: Asegúrate que el título de la alerta sea consistente entre llamadas del API.

El dashboard no carga datos (todo en skeleton)

Causa: El endpoint /api/dashboard/ devuelve error o no está disponible.
Debug: Abre DevTools → Network → busca la llamada a /api/dashboard/ → revisa el status y el body.
Causas frecuentes: Django no está corriendo, CORS bloqueado, la URL del API cambió.

El sidebar móvil no cierra / está debajo del mapa

Causa z-index: Algo tiene z-index mayor a 1100.
Causa no cierra: El overlay no está en el DOM o su onclick no está conectado.
Debug: En consola: document.getElementById('sidebar-overlay') — debe existir.

Error "Cannot read properties of null" en consola

Causa: El JS busca un elemento por ID que no existe en el HTML.
Fix general: Casi todo el código ya usa comprobaciones defensivas (if (el)). Si aparece este error, busca el ID mencionado en la tabla de IDs de arriba y verifica que exista en el HTML.
