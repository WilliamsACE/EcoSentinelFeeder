¿Qué es este panel?
Visión general del sistema · v1.0

Este panel es la extensión web del dispositivo EcoSentinel Feeder — el comedero físico inteligente instalado en la ciudad. La pantalla de inicio muestra el estado global de la red en 4 tarjetas KPI :

🐾 Animales detectados hoy Visitas de perros y gatos en el día.
⚖️ Comida dispensada Kilogramos totales repartidos en todos los feeders.
📡 Feeders en línea Comederos activos y reportando.
⚠️ Alertas activas Feeders con batería baja, comida agotada o sin conexión.

Debajo encontrarás un mapa mini y una tabla con los últimos 5 eventos. Haz clic en cualquier fila para ver el detalle completo.

Centro de alertas
Sección · Alertas

Cuando un feeder entra en estado crítico, el panel genera una alerta automática. La campana en la barra superior muestra un punto rojo si hay alertas pendientes.

⚠️

Se activan cuando batería, comida o agua caen por debajo del 20 % , o cuando un feeder pierde la conexión.

  • Batería crítica.
  • Comida agotada.
  • Agua baja.
  • Sin conexión.

Configuración y temas
Sección · Configuración

Gestiona la red de feeders y personaliza la apariencia del panel. El cambio de tema visual tiene 4 opciones:

  • Claro — paleta crema, ideal para uso diurno.
  • Oscuro — estilo Discord, menos fatiga de noche.
  • Dim — término medio.
  • Sistema — sigue la preferencia de tu OS automáticamente.

Sistema de diseño — CSS Variables
Theming · :root + [data-theme]

Todo el color está en custom properties CSS definidas en :root . Los temas dark y dim sobreescriben las mismas variables en [data-theme] — sin JS extra.

--cream
--sage-dark
--terracotta
--sky-dark
--charcoal
--danger
--ok
--warn
CSS 
/* Aplicar tema oscuro al HTML root */
applyTheme('dark');

/* Internamente: */
document.documentElement.setAttribute('data-theme', 'dark');

/* CSS reacciona automáticamente: */
[data-theme="dark"] {
  --cream: #1e1f22;
  --sage:  #7984f5;
  --terracotta: #a78bfa;
}

Sistema de secciones — SPA ligera
Navegación sin router

Navegación SPA manual sin router externo: cada sección es un <div class="page-section"> que se muestra u oculta con la clase active .

JavaScript 
function switchSection(id) {
  document.querySelectorAll('.page-section')
    .forEach(s => { s.classList.remove('active'); s.style.display = 'none'; });

  document.getElementById('section-' + id)
    ?.classList.add('active');

  if (id === 'mapa') setTimeout(initMapFull, 60);
}

Estructura de datos — Feeders
Contrato de datos del backend

renderFeeders(feeders) espera este contrato. El backend Django debe entregarlo exactamente para que gauges, mapa, sidebar y tabla funcionen correctamente.

JSON 
{
  "id":        "F-01",
  "name":      "Parque Central",
  "location":  "Av. Revolución 42",
  "status":    "online",     // "online" | "warn" | "offline"
  "lastSync":  "hace 2 min",
  "battery":   78,
  "foodDog":   55,
  "foodCat":   40,
  "water":     82,
  "dogsToday":  12,
  "catsToday":  7,
  "aiAccuracy": 94,
  "lat":        25.6866,
  "lng":       -100.3161
}
Campo Tipo Descripción
status string "online" | "warn" | "offline" — controla color del pin y badge
battery / foodDog / foodCat / water number 0–100. Bajo 20 activa alerta automática.
aiAccuracy number Porcentaje de confianza promedio del modelo de visión.
lat / lng number Coordenadas para el pin de Leaflet.

Sistema de notificaciones
notifStore · renderNotifications

Las alertas viven en el array global notifStore[] . renderNotifications() actualiza el panel lateral, el badge de la campana y el badge de la sección Alertas.

JavaScript 
if (f.battery < 20)
  notifStore.push({ type: 'danger', msg: `🔋 ${f.id} batería crítica` });

if (f.foodDog < 20)
  notifStore.push({ type: 'warn', msg: `🦴 ${f.id} comida perro baja` });

if (f.status === 'offline')
  notifStore.push({ type: 'danger', msg: `📶 ${f.id} sin conexión` });

renderNotifications();

Sistema de tema — applyTheme()
localStorage · prefers-color-scheme

applyTheme(mode, save?) centraliza el theming en 4 modos: 'light' , 'dark' , 'dim' y 'auto' . La preferencia se persiste en localStorage con la clave 'eco-theme' .

1
Leer preferencia guardada
localStorage.getItem('eco-theme') || 'auto' al iniciar la página.
2
Resolver modo
Si es 'auto' , evaluar window.matchMedia('(prefers-color-scheme: dark)').matches .
3
Aplicar al DOM
document.documentElement.setAttribute('data-theme', resolvedMode) — el CSS hace el resto.
4
Actualizar tiles del mapa
En modo dark/dim se aplica filter: invert + hue-rotate a las capas de Leaflet.

Gauges circulares — SVG puro
stroke-dashoffset · sin librerías

Gauges renderizados con SVG inline via stroke-dashoffset . Radio 26 → circunferencia 163.4 . Sin librerías externas.

JavaScript 
// Convierte % a stroke-dashoffset
const pct = (v) => Math.round(163.4 * (1 - v / 100));

// Batería al 78 %  →  offset ≈ 36
// Comida al 0 %   →  offset = 163.4

`<circle stroke="${sBat}"
  stroke-dasharray="163.4"
  stroke-dashoffset="${pct(f.battery)}"/>`

Integración Leaflet — Mapa interactivo
Lazy init · Dark tile filter

El mapa se inicializa lazily con 60 ms de delay al activar la sección, para que el DOM esté visible antes de que Leaflet calcule dimensiones.

🌙 Tema oscuro En modo dark/dim se aplica filter: invert(100%) hue-rotate(180deg) brightness(0.85) a las capas del mapa — sin recargar tiles.
📍 Pins dinámicos Generados como divs con clase .feeder-pin y actualizados mediante _refreshMapPins(feeders) .
💬 Popups personalizados Clase esf-popup con CSS inyectado para seguir el design system del panel.
